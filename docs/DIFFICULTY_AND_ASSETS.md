# 游戏难度分级与图片制作规范（v1）

> 适用于 Spot Odd One 全品类（Pack）  
> 与单品类文档（`CAT_THEME.md` / `DOG_THEME.md` / `FAIRY_THEME.md` 等）配合使用  
> **共性规则以本文为准；品类文档只写该包特有曲线与角色卡**

---

## 1. 三种难度模式（Pack 级）

每个主题包在 Hub 上归属 **简单 / 中等 / 困难** 之一。  
玩家选包 = 选难度；包内再按关卡段递进，不在包内再切模式。

| 模式 | 英文 | 认知负担 | 适合谁 | 已规划 / 已上线品类 |
|------|------|----------|--------|---------------------|
| **简单** | Easy | 低 → 中 | 全年龄、快速上手 | Emoji · Monsters · Cats · Dogs |
| **中等** | Medium | 中 → 中高 | 玩过简单包、喜欢观察细节 | **Fairy（仙女）** ← 下一品类 |
| **困难** | Hard | 高 | 追求挑战、能接受微差与动效 | （后续品类，如 Micro / Chaos 等） |

### 1.1 模式差异总览

| 维度 | 简单 Easy | 中等 Medium | 困难 Hard |
|------|-----------|-------------|-----------|
| 建议总关数 | **120** | **80–100**（仙女默认 **85**） | **60–80** |
| 开局可辨识度 | 一眼能分 | 要扫一眼细节 | 需仔细对比 |
| 差异尺度 | 物种 / 品种 / 大造型 | 部件级（发色、衣服、道具） | 微部件 + 干扰 |
| 网格 | 3×3 → 4×4 → 5×5 | 以 4×4 为主，后期 5×5 | 以 5×5 为主 |
| 变换（旋转/缩放/色相） | 中后期可用、偏明显 | 慎用；优先「部件真不同」 | 可与微差叠加 |
| 动效（teleport 等） | 仅地狱段、且差异够明显 | 仅最高难段 | 可更早、更密 |
| 美术 | 插画或 emoji；插画需统一风格 | **必须精致手绘卡通** | 精致手绘 + 极高一致性 |
| 限时 | 全关 **10s** | 全关 **10s** | 全关 **10s**（难度不靠加时） |

## 2. 各模式 · 关卡设置

共性规则（三模式相同）：

- 每关恰好 **1** 个 Odd One；点错或超时即失败。  
- 续关 / 分享规则见 `FAIL_SHARE_AD_SPEC.md`。  
- 进度按 **主题包** 分存（`themes.<packId>`）。

### 2.1 简单模式（Easy）— 关卡模板

参考已上线：Cats / Dogs / Monsters（120 关）。

| 段 | 建议关数 | 网格 | 差异设计 | FX |
|----|----------|------|----------|-----|
| 入门 | 1–30 | 3×3 | **完全不同角色**（如猫群里一只兔子） | 无 |
| 巩固 | 31–50 | 3×3→4×4 | 仍是「不同角色」，扩大角色池 | 无 |
| 中段 | 51–80 | 4×4 | **同大类不同亚类**（品种 / 剪影分组） | 无 |
| 高段 | 81–100 | 5×5 | 同角色：明显色差 / 装饰 twin | 无 |
| 地狱 | 101–120 | 5×5 | 更强差异或 twin；偶数关可 `teleport` | teleport（可选） |

**设计原则：** 前 50 关禁止「只有 5° 旋转」这类肉眼难辨差异；对称角色禁止纯镜像。

### 2.2 中等模式（Medium）— 关卡模板

以 **Fairy（仙女）** 为样板；总关建议 **85**（可在区间内微调）。

| 段 | 建议关数 | 难度感 | 差异设计 | 网格 | FX |
|----|----------|--------|----------|------|-----|
| **简单段** | **15**（区间 10–20） | 轻松 | 一群仙女里，有一个 **明显不一样**（发型轮廓 / 体型 / 主色整块不同，扫一眼可见） | 3×3→4×4 | 无 |
| **中等段** | **40**（区间 30–50） | 主力 | **同一仙女造型体系**，Odd 仅 **某一部位** 不同：发色 · 衣服 · 手持道具 · 头饰/翅膀等装饰 | 4×4 | 无 |
| **过渡段** | **10** | 略放松 | 比中等段 **更易**（差异稍大或回到「较明显部件差」），给玩家喘口气，再进高难 | 4×4 | 无 |
| **高难段** | **20** | 难 | **更小细节**（耳环形状、袖口花纹、鞋扣、翅膀尖端颜色等）+ 图片可 **移动/换位**（`teleport` 或等价） | 5×5 | teleport（建议过半开启） |

合计默认：15 + 40 + 10 + 20 = **85 关**。

**中等模式禁忌：**

- 不要用 emoji 混进插画关。  
- 中等段不要靠 CSS 轻微 `hue` 冒充「发色不同」——发色差应是 **画出来的不同资产**（或明确 twin 图）。  
- 过渡段禁止突然上微差 + 换位双杀。

### 2.3 困难模式（Hard）— 关卡模板（预留）

| 段 | 建议关数 | 差异设计 | 网格 | FX |
|----|----------|----------|------|-----|
| 热身 | 1–10 | 仍可读的部件差（比中等高难段略易） | 4×4 | 无 |
| 主体 | 11–40 | 微差 twin；同构图极高一致 | 5×5 | 轻 wobble 可选 |
| 压迫 | 41–60+ | 微差 + 换位 / 抖动；可叠加弱色差 | 5×5 | teleport 为主 |

困难包上线前需单独评审「可辨识性」与误触率。

## 3. 图片制作规则（全插画品类强制）

### 3.1 画布与导出

| 项 | 规则 |
|----|------|
| 尺寸 | **1024 × 1024 px**（正方形，1:1） |
| **入库格式** | **仅 WebP（`.webp`）** — 游戏与仓库正式素材只认此格式 |
| 背景 | **纯白**或透明；同一品类内必须统一 |
| 主体 | 单角色、居中；四周留 **约 10–12%** 安全边距（防旋转裁切） |
| 禁止 | 文字、水印、Logo、复杂场景、多角色同框 |
| 命名 | 小写 + 品类前缀，如 `fairy_f01.webp`、`dog_d01b.webp` |
| 目录 | `apps/web/public/<pack>/` |

### 3.1.1 图片格式限定（强制）

| 阶段 | 允许格式 | 说明 |
|------|----------|------|
| **最终入库 / 上线** | **仅 `.webp`** | 代码 `IMAGE_EXT = "webp"`；Hub、关卡、预览一律 WebP |
| 制作中间稿 | PNG（`.png`）可临时使用 | 画师/生成工具常用；**不得**以 PNG 提交进 `public/` 长期保留 |
| **禁止入库** | JPEG / JPG、GIF、BMP、TIFF、AVIF、ICO 等 | 不支持有损 JPG 白边、动图 GIF、多帧等 |

转换要求：

1. 中间稿若为 PNG，在 `apps/web` 执行：`npm run png:webp`（默认 quality=85）  
2. 确认无误后：`npm run png:webp:delete` 删除 PNG，避免双份体积  
3. 可选：`--quality=80~90`；需透明通道时保持 alpha  
4. 单张 WebP 建议 **小于 200–400KB**；超出先压再转  

校验清单（入库前）：

- [ ] 扩展名为 `.webp`  
- [ ] 像素 **1024×1024**  
- [ ] `public/` 对应目录下 **无同名 `.png`**  
- [ ] 游戏内能正常显示（含透明区域）  

### 3.2 风格

| 项 | 规则 |
|----|------|
| 画风 | **手绘感卡通 / kawaii chibi**，干净描边 + 柔和赛璐璐着色 |
| 气质 | 休闲益智、全年龄；**仙女包偏女性向**（柔和配色、优雅可爱，非写实、非性感擦边） |
| 一致性 | 同一品类光源、描边粗细、眼睛高光语言必须统一 |
| 辨识 | 简单/中等「明显不一样」靠 **剪影或主色块**；中等部件差靠 **局部设计**，不要整只重画到认不出是同一角色 |

### 3.3 Twin / 变体规则

| 用途 | 做法 |
|------|------|
| 部件差（发色、衣服、道具、装饰） | **单独出图**（如 `F01` vs `F01_hair_pink` 或 `F01b`），禁止只靠运行时滤镜冒充关键差异 |
| 地狱微差 | 可出 twin，或极克制滤镜；若用滤镜，必须真机验收「多数人 10s 内可辨」 |
| 装饰示例 | 铃铛、蝴蝶结、项圈、耳环、魔杖、翅膀尖端颜色 |

### 3.4 生成 / 外发提示词要求

提示词必须写明：

1. `1024x1024` / `square composition`  
2. `plain white background`（或统一透明）  
3. `single character centered`  
4. 品类风格锚点（见各 `*_PROMPTS.md`）  
5. 负面：写实、恐怖、文字、多角色、复杂背景  

出图后用工具校验像素为 **1024×1024**，再转为 **WebP** 入库。

### 3.5 性能与工程约定

- 单包主图建议控制在 **8–16 张主角色 + 若干 twin**。  
- Hub / 开始页预览必须用 **本包插画 WebP**，禁止 emoji 示意混用。  
- 扩展名常量：`apps/web/src/game/imageExt.ts` → `IMAGE_EXT = "webp"`（改格式时只改此处 + 全量转码）。
